问题描述

(原题链接：LeetCode | 44. Wildcard Matching)

请实现支持’?’和’*’字符的通配符匹配功能。详细说明如下：

  • ‘?’可以匹配任意的单个字符
  • ‘*’可以匹配任意的字符序列(包括空字符序列)

匹配规则要求能覆盖整个待匹配的字符串，而不是其中的一部分。

函数的原型为：bool isMatch(const char *s, const char *p)，其中s表示待匹配的字符串，p表示匹配规则，返回值表示是否匹配成功。例如：

isMatch("aa","a") → false
isMatch("aa","aa") → true
isMatch("aaa","aa") → false
isMatch("aa", "*") → true
isMatch("aa", "a*") → true
isMatch("ab", "?*") → true
isMatch("aab", "c*a*b") → false

解题思路

题目中关于’*‘的匹配规则是经过了简化，因此降低了题目的难度。

这道题目可以通过回溯的办法来解决。设置两个分别指向待匹配字符串和匹配规则的指针，初始化为0，然后按照如下规则前进：

  1. 如果两个指针指向的字符匹配成功，则一同前进；
  2. 如果指向匹配规则的指针对应的字符为’*‘，则记录下匹配规则中最近一次出现’*‘的位置(记为ip_star)，以及此时指向待匹配字符串指针的位置(记为is_star)，然后跳过’*‘继续进行匹配；
  3. 如果两个指针指向的字符匹配失败，则指向匹配规则的指针返回ip_star+1的位置，指向待匹配的字符串的指针返回is_star+1的位置，并且将is_star指向的位置向后挪一位；
  4. 如果两个指针指向的字符匹配失败，同时没有在匹配规则中出现过’*’，那么一定匹配失败。

按照如上步骤移动指针，直到指向待匹配字符串的指针达到字符串的终点，如果指向匹配规则的指针也同时到达终点或者剩余的字符全部为’*’，那么匹配成功，否则匹配失败。

以上的描述略显抽象，用一句话总结：待匹配字符串和匹配规则逐个字符匹配，如果失败就返回最近遇到’*’的位置重新尝试。这也是为什么叫做回溯法的原因。

时间复杂度分析：

  • 最佳情况下为O(max(n, m))
  • 最差情况下为O(n*m)

其中n和m分别表示待匹配字符串和匹配规则字符串的长度。
